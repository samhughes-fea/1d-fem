# Job layout

Summary of what each job is for. Use this when interpreting `post_processing/results` and when running verification visualisers (e.g. Roark vs FEA, deformation convergence).

## Naming convention: job_XXXX_nX

Job directories use **job_XXXX_nX** where **XXXX** is the base job code (e.g. 0000, 0005) and **X** is the number of mesh elements (e.g. n10, n100). Examples: `job_0000_n10`, `job_0005_n100`, `job_0005_n4`. Verification scripts discover results by parsing this name and resolve the job input dir (e.g. `jobs/job_0005_n100/grid.txt`). For convergence studies, multiple mesh densities exist per load type (e.g. job_0005_n4, job_0005_n8, job_0005_n16, job_0005_n100).

## Point load – Euler–Bernoulli, three positions (job_0000, job_0001, job_0002)

Same cantilever, Euler–Bernoulli element; point load position varies (end, midspan, quarter). Current variants: job_0000_n10, job_0001_n10, job_0002_n10; mesh-density variants job_0000_n4, n8, n16, n25, n50, n100 (and similarly for job_0001, job_0002) are generated by `pre_processing/mesh_library/create_point_load_mesh_variants.py` (run from repo root).

| Job           | Load position   | Formula / x   |
|---------------|------------------|---------------|
| job_0000_n10  | End (tip)        | P(x=L)        |
| job_0001_n10  | Midspan          | P(x=L/2)      |
| job_0002_n10  | Quarter-point    | P(x=L/4)      |

Use these to verify Euler–Bernoulli tip/midspan/quarter deflection vs analytical (e.g. Roark point-load formulas with different `a`).

## End load – different element types (job_0003, job_0004)

Same cantilever with tip (end) point load; only the beam element type changes. Current variants: job_0003_n10, job_0004_n10.

| Job           | Element type              | Load        |
|---------------|---------------------------|------------|
| job_0003_n10  | TimoshenkoBeamElement3D   | End (tip) point load |
| job_0004_n10  | LevinsonBeamElement3D     | End (tip) point load |

Use these to compare element formulations (Timoshenko vs Levinson) under the same end load.

## Truss and Bar elements

| Element type        | Carries                    | Use case                          |
|---------------------|----------------------------|-----------------------------------|
| TrussElement3D      | Axial, transverse, torsion | Pin-jointed + shear + torsion     |
| BarElement3D        | Axial, torsion only        | Axial + torsion, no transverse    |

Both use 6 DOF per node and return 12×12 K_e, 12×1 F_e. Register in the [Element] file as `TrussElement3D` or `BarElement3D`.

**Optional job**: `job_bar_single` – one Bar element, fixed at node 0, axial point load at node 1; for pipeline check.

## Distributed loads – Euler–Bernoulli (job_0005, job_0006, job_0007)

Same cantilever, Euler–Bernoulli element; load type varies (UDL, triangular, parabolic). Mesh-density variants: job_0005_n4, job_0005_n8, job_0005_n16, job_0005_n25, job_0005_n50, job_0005_n100 (and similarly for job_0006, job_0007) for convergence studies.

| Job            | Load type   | Formula / profile        |
|----------------|-------------|---------------------------|
| job_0005_n*    | UDL         | q(x) = w (constant)       |
| job_0006_n*    | Triangular  | q(x) = w·(x/L)            |
| job_0007_n*    | Parabolic   | q(x) = w·(x/L)²           |

Use these for Roark vs FEA verification of distributed-load formulas (e.g. `post_processing/verification_visualisers/roarks_formulas`) and for deformation convergence (error vs n_elements) via `deflection_tables/distributed_load_convergence.py`. The GCI and Richardson extrapolation report (Roark vs FEM tip deflection/rotation for jobs 0,1,2,5,6,7) is produced by `deflection_tables/gci_richardson_roark_report.py` and requires result dirs for n25, n50, n100. Mesh-density variants (job_0005_n4, n8, n16, n25, n50, n100, etc.) are generated by `pre_processing/mesh_library/create_distributed_mesh_variants.py` (run from repo root).

## jobs/bin

Duplicate/canonical definitions for some of the above (e.g. UDL, triangular, parabolic) also live under `jobs/bin` (job_0004, job_0005, job_0006 there map to UDL, Triangular, Parabolic). Verification scripts may reference either `jobs/` or `jobs/bin` for job input files.
